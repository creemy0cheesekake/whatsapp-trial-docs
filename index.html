Get Started

1. Create a Twilio Account

Click on this link and click "Sign up for free" to create a Twilio account. After creating the account, you should be in the Twilio console

2. Get a trial number

In the middle of the screen, you should see a button that says "Get a trial number". Click on that, then there should be a popup window. Click on Choose this number, then click on Done. Keep note of the phone number.

3. Check Account SID and Auth Token

In the middle of the screen, there should be two boxes that say "ACCOUNT SID" AND "AUTH TOKEN".

Keep note of both of these values and store them securely. (Click on "Show" to reveal the Auth Token).

4. Join Twilio trial

On the left, you should see an option that says "Messaging". Click on that, then click on Settings, then click on WhatsApp sandbox settings. There should be a bolded text in the middle of the screen that says join <xxx-xxx>. Right before that, there should be a phone number. Open WhatsApp on your phone and send the join <xxx-xxx> code to the number that was there.

Note: This step is not needed for use in production.

5. Calling the API

Everything should be all set at this point, the only thing left to do is to call the API. The API has 2 endpoints, one to send a message, and another to get the status of the latest sent message.

In order to send the message, a POST request should be sent to / of the provided API endpoint. This POST request should have the following data sent in x-www-form-urlencoded format (fill in the placeholders with the real data):

					
{
	sid: <ACCOUNT SID>
	from: <TRIAL PHONE NUMBER>
	to: <RECIPIENT'S PHONE NUMBER>
	message: <MESSAGE TO SEND>
	media: <URL OF IMAGE>
}

and should be sent with the following header:{ method: 'POST', Authorization: 'Bearer <AUTH TOKEN>' }

To check the status of the latest sent message, send a GET request to /msg/status/info?sid=<ACCOUNT SID> with the following header: headers: {Authorization: 'Bearer <AUTH TOKEN>'}


At this point, everything should be working!
